LOISIRS 74 — DEPLOY CHECKLIST (5 LANGUAGES)
===========================================

ARCHITECTURE
------------
- FR is the master language. All content authored in FR first.
- 5 languages: fr, en, de, it, es
- URL scheme:
    /                       FR homepage        (index.html)
    /<lang>/                lang homepage      (en/index.html, etc.)
    /<slug>                 FR lieu page       (domaine-du-tornet.html)
    /<lang>/<slug>          lang lieu page     (en/domaine-du-tornet.html)
- Netlify pretty_urls handles .html → extensionless.

SOURCES OF TRUTH
----------------
- lieux.json             What appears on the homepage grid (5-lang names)
- Json/<slug>.json       Everything for ONE lieu, unified across 5 langs
- index.html             FR homepage template (translated by translate_index.py)
- loisirs74-template-v3.html   Lieu page template (localized by render-v3.py)

PER-LIEU WORKFLOW
-----------------
1. Write unified Json/<slug>.json with i18n.fr/en/de/it/es
2. python3 render-lieu.py <slug>        → 5 HTML pages
3. Add entry to lieux.json              → bump with 5-lang i18n
4. python3 build-index.py               → regenerates all 5 indexes
5. Add 5 <url> blocks to sitemap.xml with hreflang alternates
6. git commit + push → Netlify auto-deploys

HOMEPAGE-ONLY CHANGES
---------------------
1. Edit index.html (FR)
2. python3 build-index.py               → regenerates all 5 indexes
   (or python3 translate_index.py if only translator dicts changed)

WHEN SPANISH STRING IS MISSING
------------------------------
Symptom : text appears in French in es/ pages
Fix :
  - If it's an EDITORIAL string (homepage copy): add to ES_ADDITIONS in translate_index.py
  - If it's a UI string (inside a lieu page, like "Back to top"): add to TEMPLATE_STRINGS or UI_STRINGS in render-v3.py
  - Rerun build-index.py + render-lieu.py <slug> for affected files

VALIDATION BEFORE DEPLOY
------------------------
- All HTMLs have balanced tags
- Each /<lang>/* file has <html lang="<lang>"> matching its folder
- No FR leaks in non-FR files (canary strings: "Comment y aller", "Sur cette page", "Où manger", "Signaler une info", "Questions fréquentes")
- sitemap.xml parses as valid XML with 5 hreflangs + x-default per URL
